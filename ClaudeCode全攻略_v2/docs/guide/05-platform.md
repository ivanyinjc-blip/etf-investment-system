# 入口地图：选对适合你的入口

Claude Code 支持四种不同的使用入口，每个入口都有独特的定位和适用场景。选择对的入口，能让你的工作效率翻倍。

---

## 四大入口对比表

| 入口 | 适合场景 | 优势 | 劣势 |
|------|---------|------|------|
| **VSCode 插件** | 主力开发、代码解释、Inline Edit | 原生融合、Inline Edit、无缝切换 | 需安装 VSCode |
| **JetBrains** | Java/Kotlin/大型项目、深度集成 | 深度集成、框架理解强 | 性能占用较大 |
| **CLI 命令行** | 脚本自动化、Server、多工具联动 | 最高自由、管道组合、可编程 | 无 UI，需记命令 |
| **Cloud 网页** | 快速试用、多设备、零配置 | 即开即用、跨平台 | 功能受限、需联网 |

---

## 各入口详解

### VSCode 插件

**定位**：最推荐的开发入口，深度融入 IDE 体验。

**安装方式**：

1. 打开 VSCode
2. `Ctrl+Shift+X` 打开扩展市场
3. 搜索 **"Claude Code"**
4. 点击安装

**核心功能**：

| 功能 | 说明 |
|------|------|
| Inline Chat | 在代码行内直接对话（`Cmd+I` 触发） |
| Side Panel | 侧边栏完整对话窗口 |
| Inline Diff | 直接在代码中显示修改对比 |
| 快捷键绑定 | 可自定义触发快捷键 |

**适用人群**：

- ✅ 主力开发用 VSCode 的工程师
- ✅ 需要边写代码边解释的开发者
- ✅ 喜欢 Inline Edit（内联编辑）的人

> **Tip**：VSCode 插件和 CLI 底层是同一个工具，配置（API Key、权限模式）是共享的，可以同时使用。

---

### JetBrains（IntelliJ IDEA / PyCharm / WebStorm 等）

**定位**：面向 Java/Kotlin/Python 大型项目的深度集成。

**安装方式**：

1. 打开 JetBrains IDE
2. `Settings` → `Plugins` → 搜索 **"Claude"**
3. 点击安装并重启 IDE

**核心功能**：

| 功能 | 说明 |
|------|------|
| 内嵌对话窗口 | IDE 侧边栏直接对话 |
| 代码补全增强 | AI 辅助代码补全 |
| 项目理解 | 深度理解项目结构和依赖 |
| 框架集成 | Spring/Pytest/Django 等框架感知 |

**适用人群**：

- ✅ Java/Kotlin 大型项目开发者
- ✅ 依赖 JetBrains 深度功能（如重构、调试）的工程师
- ⚠️ 注意：JetBrains 插件资源占用较高，低配机器慎用

---

### CLI 命令行

**定位**：最高自由度的入口，适合自动化、脚本化、Server 端使用。

**启动方式**：

```bash
# 基本启动
claude

# 指定模式
claude auto          # Auto 模式
claude plan          # Plan 模式
claude browse        # Browse 模式

# 指定 Session
claude --session "my-task"

# 非交互模式（静默执行）
claude --print "请解释 src/app.py 的逻辑"
```

**与外部工具联动**：

```bash
# 结合 Git
claude | git commit -F -

# 结合文件处理
cat error.log | claude --print "分析这个错误日志"

# 多工具管道
find . -name "*.py" | xargs claude --print "给这些文件加英文注释"
```

**适用人群**：

- ✅ 需要自动化脚本的 DevOps/后端工程师
- ✅ Server 环境（无 GUI）用户
- ✅ 喜欢管道式编程的极客
- ✅ 需要和其他 CLI 工具（jq/grep/awk）配合使用

---

### Cloud 网页

**定位**：零配置的快速试用入口，跨平台跨设备。

**访问地址**：https://claude.ai/claude-code

**特点**：

- ✅ 无需安装，打开浏览器就能用
- ✅ 支持手机/平板临时使用
- ✅ 配置自动同步
- ⚠️ 功能相对桌面版有所限制
- ⚠️ 依赖网络连接

**适用人群**：

- ✅ 首次体验，想快速感受 Claude Code
- ✅ 在别人的电脑上临时使用
- ✅ 多设备（Windows/Mac/Linux）无缝切换
- ⚠️ 注意：网页版无法使用本地文件工具

## 多入口切换注意事项

如果你同时使用多个入口，以下事项需要注意：

### 1. 配置是共享的

```
API Key / OAuth 认证 → 所有入口共享
权限模式设置 → 所有入口共享
Session 历史 → 各入口独立存储
```

### 2. Session 不自动同步

```bash
# VSCode 里的 Session
# → 存储在 ~/.config/claude-code/sessions/

# CLI 里的 Session
# → 存储在 ~/.claude-code/sessions/

# 不会自动互通，需要手动恢复：
claude sessions list   # 查看 Session ID
claude --resume <id>   # 恢复指定 Session
```

### 3. 文件路径的差异

| 入口 | 工作目录 | 文件路径解析 |
|------|---------|------------|
| VSCode 插件 | 当前打开的 Workspace | 相对于 Workspace |
| JetBrains | 项目根目录 | 相对于项目根目录 |
| CLI | 当前终端目录 | 相对于执行命令的目录 |
| Cloud 网页 | 需手动指定 | 需上传文件 |

### 4. 工具权限可能不同

```
VSCode/JetBrains 插件：可访问 IDE 内置的文件浏览器
CLI：直接访问文件系统
Cloud 网页：沙盒环境，文件操作受限
```

## 最佳实践：CLI 为主，IDE 辅助

### 推荐工作流

```
日常开发 → VSCode 插件（主力）
    ↓
遇到复杂任务 / 需要管道组合 → CLI
    ↓
临时在外 / 快速试用 → Cloud 网页
    ↓
Java/Kotlin 大项目 → JetBrains
```

### 场景举例

| 场景 | 推荐入口 | 原因 |
|------|---------|------|
| 边写代码边让 AI 解释 | VSCode 插件 | Inline Chat 最方便 |
| 批量给 100 个文件加注释 | CLI | 管道 + xargs 最快 |
| 在地铁上临时看代码 | Cloud 网页 | 手机就能打开 |
| Java 微服务重构 | JetBrains | 项目理解最深度 |
| 写自动化部署脚本 | CLI | 可编程、自由度高 |
| 首次体验 Claude Code | Cloud 网页 | 零配置 |

---

## 总结

> **一句话推荐**：
> - **VSCode 用户** → 装插件，用 Inline Chat
> - **JetBrains 用户** → 装插件，用 Side Panel
> - **自动化/Server** → CLI 命令行
> - **快速试用** → Cloud 网页

多入口可以并存，根据任务自由切换，这才是 Claude Code 的正确打开方式。🚀
