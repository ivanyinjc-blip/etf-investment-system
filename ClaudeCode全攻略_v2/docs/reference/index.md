## 第三方教程推荐标准

### 如何判断教程是否过时

**🚩 危险信号（教程可能过时）**

| 信号 | 说明 |
|------|------|
| 缺少日期 | 没有发布/更新日期 |
| 版本号旧 | 还在说 "v0.x" 或 "Beta" |
| 命令报错 | 复制命令执行报语法错误 |
| 界面不同 | 截图和实际界面差异大 |
| 评论区有 "失效" 反馈 | 其他人也在说不行 |

**✅ 健康信号（教程相对可靠）**

| 信号 | 说明 |
|------|------|
| 有明确日期 | 发布和最后更新时间都标注 |
| 版本一致 | 文档版本与你的 Claude Code 一致 |
| 有官方背书 | 作者是 Anthropic 员工或社区认可 |
| 持续更新 | 最近 3 个月有更新记录 |
| 评论区活跃 | 有问题能得到解答 |

### 推荐资源筛选标准

```
1. 作者背景：Anthropic 官方 > 社区知名贡献者 > 普通用户
2. 更新频率：优先选最近 6 个月内更新的
3. 内容质量：有代码示例 > 只有文字说明
4. 社区反馈：评论多且正向 > 评论少或负面
```

## 版本更新追踪方法

### 方法一：GitHub Releases（推荐）

```bash
# 订阅 GitHub releases
# 1. 打开 https://github.com/anthropics/claude-code/releases
# 2. 点击 "Watch" → "Releases only"
# 3. 选择 "Notifying on Discord"（如果你在 Discord）
```

### 方法二：官方博客订阅

```bash
# 1. 打开 https://www.anthropic.com/news
# 2. 滚动到页面底部
# 3. 输入邮箱订阅更新
```

### 方法三：RSS 订阅

```xml
<!-- RSS 订阅地址 -->
https://github.com/anthropics/claude-code/releases.atom
```

### 版本号规则

```
claude-code 采用语义化版本：major.minor.patch

例：v1.2.3
- major（1）：重大架构变更，可能不兼容旧用法
- minor（2）：新功能，向后兼容
- patch（3）：Bug 修复，向后兼容
```

### 更新时必看的内容

| 内容 | 为什么看 |
|------|---------|
| Breaking Changes | 可能影响现有工作流 |
| New Features | 发现提升效率的新能力 |
| Bug Fixes | 解决你可能遇到的坑 |
| Deprecations | 即将移除的功能，提前适配 |
