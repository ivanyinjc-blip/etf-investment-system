# 团队落地 Playbook

> 让 Claude Code 在团队中从「个人工具」变成「团队基础设施」的完整路径

---

## 三个落地阶段

### 阶段一：个人提效（1-2 周）

**目标**：让 1-2 个成员先跑通，建立信心

**做什么**：
- 选择一个高频、低风险场景开始（如生成文档、自动修复 CI）
- 记录使用心得和效果数据
- 整理遇到的问题和解决方案

**成功标准**：
- ✅ 单人完成至少 3 个实际任务
- ✅ 有具体的「节省时间」数据
- ✅ 发现 2-3 个可复制的任务模板

**负责人**：个人使用者（自愿报名最佳）

---

### 阶段二：项目试点（2-4 周）

**目标**：在一个项目中验证完整流程，沉淀规范

**做什么**：
- 选择一个试点项目（中等复杂度，有代表性）
- 制定项目级 AGENTS.md（规则文档）
- 标准化任务描述模板
- 沉淀第一批成功案例
- 定期同步（每周 15 分钟站会）

**成功标准**：
- ✅ 试点项目中有 5+ 个任务使用 Claude Code 完成
- ✅ 有书面的规范文档（AGENTS.md + 任务模板）
- ✅ 团队成员对效果有共识

**负责人**：项目负责人 + 试点成员

---

### 阶段三：团队推广（持续）

**目标**：标准化复制，扩大适用范围

**做什么**：
- 整理并发布团队规范文档库
- 组织分享会（让试点成员分享经验）
- 收集常见问题，编写排障手册
- 制定贡献路径，欢迎新成员补充案例
- 定期回顾效果指标，持续优化

**成功标准**：
- ✅ 80%+ 成员了解并尝试使用
- ✅ 有 3+ 个稳定运行的工作流
- ✅ 有量化的价值数据（提效比例、任务完成率）

**负责人**：团队负责人 + 技术推广者

---

## 团队规范文档结构

一个标准的 Claude Code 团队知识库应包含以下文档：

```
team-claude-code/
├── AGENTS.md              # 项目级 AI 使用规则
├── 任务模板/              # 标准化任务描述
│   ├── 新功能开发.md
│   ├── Bug修复.md
│   ├── 文档生成.md
│   └── 代码审查.md
├── 案例库/                # 沉淀的成功案例
│   ├── index.md
│   ├── 案例1-xxx.md
│   └── 案例2-xxx.md
├── 排障手册.md            # 常见问题与解决方案
└── 贡献指南.md            # 新成员加入指南
```

---

### AGENTS.md 示例

```markdown
# [项目名] AI 使用规范

## 我们的共识
Claude Code 是我们的助手，不是替代者。我们用它处理重复性工作，
但关键决策、架构设计、业务逻辑仍由人负责。

## 允许做的事
- ✅ 生成文档、注释、测试代码
- ✅ 自动化脚本编写
- ✅ 代码重构和优化建议
- ✅ 日志分析和问题定位

## 禁止做的事
- ❌ 直接提交到 main/master 分支
- ❌ 修改生产环境配置
- ❌ 对外发送任何内容（邮件、消息、帖子）
- ❌ 删除文件（只移动到 trash）

## 提交规范
- Claude Code 的每次操作需人工 Review
- Commit message 需明确说明改了什么、为什么改
- 涉及多人协作的改动需通知相关人

## 上下文管理
- 复杂任务使用 CLAUDE.md 注入背景
- 多步骤任务结束后清理上下文，避免信息混乱
- 重要决策记录到决策日志中
```

## 团队负责人职责清单

- [ ] **宣导**：让全员知道 Claude Code 的存在和价值
- [ ] **选人**：找到 1-2 个早期采用者带头
- [ ] **定规范**：制定 AGENTS.md，明确允许/禁止事项
- [ ] **造场景**：识别团队高频任务，推广适用场景
- [ ] **解阻力**：及时响应「不好用」「不信任」的反馈
- [ ] **沉淀**：推动案例库和排障手册的建立
- [ ] **量化**：定期统计使用数据，证明价值
- [ ] **迭代**：根据反馈持续优化规范和工作流

---

## 推广阻力与应对策略

| 阻力类型 | 典型声音 | 应对策略 |
|---------|---------|---------|
| **不信任** | 「AI 写的代码能信吗？」 | 强调人工 Review 流程；展示成功案例数据 |
| **不知道怎么用** | 「我不知道问什么」 | 提供任务模板和示例对话 |
| **觉得没用** | 「我直接写更快」 | 找到他最高频的重复任务，专门演示 |
| **担心安全** | 「代码会不会泄露？」 | 说明本地运行原理，数据不外传 |
| **工具太多** | 「学不动了」 | 聚焦一个场景，用起来再说 |
| **领导不支持** | 「这东西靠谱吗？」 | 拿出量化数据（节省时间、减少错误） |

## 成功案例分享

### 案例：某开发团队 CI 修复自动化

**背景**：团队 8 人，CI 频繁失败，每次人工排查平均 2 小时

**做法**：
1. 先在 1 个项目试点 GitHub Actions 自动修复
2. 沉淀了 YAML 配置模板
3. 推广到所有后端项目

**数据**：
- CI 修复平均时间：2h → 15min
- 团队每月节省：约 20 人力小时
- 自动修复成功率：78%（复杂问题仍需人工）

**分享方式**：团队内部分享会（30 分钟），现场演示 + 数据展示

### 案例：文档团队内容生产提效

**背景**：文档团队 3 人，每月产出 20+ 篇技术文档

**做法**：
1. Claude Code 生成文档初稿
2. 人工审核 + 润色
3. 沉淀文档模板库

**数据**：
- 单篇文档平均耗时：3h → 1h
- 提效比例：66%
- 内容一致性提升（模板统一风格）

## 新成员加入指南

### 快速上手（第一天）

1. **阅读 AGENTS.md**（15 分钟）
   - 了解团队规范和红线

2. **本地安装 Claude Code**（10 分钟）
   ```bash
   # 安装
   npm install -g @anthropic-ai/claude-code
   
   # 验证
   claude --version
   ```

3. **完成第一个任务**（30 分钟）
   - 从任务模板库选一个简单任务
   - 在有经验的成员指导下完成

4. **加入案例库**（随时）
   - 成功完成任务后，记录到案例库
   - 遇到问题后，记录到排障手册

### 常见问题

**Q: 我遇到的问题在排障手册里没有怎么办？**
A: 在团队群里提问，同时记录下来，解决后补充到手册。

**Q: 我不确定这个任务能不能用 Claude 怎么办？**
A: 问团队负责人，或者直接试试——只要不涉及禁止事项都可以尝试。

**Q: Claude 的回答我不确定对不对怎么办？**
A: 正常，AI 会出错。关键是人工 Review 环节，任何不确定的地方都要验证。
