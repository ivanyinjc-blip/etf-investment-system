# Claude Code × Obsidian：本地 AI 知识库搭建

> 把你的 Obsidian 笔记库变成可以自然语言对话的私人知识助理

---

## 场景

- 你积累了大量 Obsidian 笔记（项目文档、学习笔记、会议记录）
- 想让 AI 理解这些笔记，用自然语言查询
- 想让 AI 帮你生成摘要、标签、链接关系
- 不希望笔记上传到云端，保持本地隐私

---

## 工具栈

| 工具 | 用途 | 版本要求 |
|------|------|---------|
| Claude Code CLI | AI 对话核心 | 最新版 |
| Obsidian Vault | 本地笔记存储 | 任意版本 |
| 文件系统 | 读取 .md 文件 | - |

---

## 核心步骤

### Step 1：在 Obsidian 库目录初始化 Claude Code

```bash
# 进入你的 Obsidian 笔记库目录
cd ~/Obsidian/MyVault

# 初始化 Claude Code 项目
claude --setup
```

初始化后，该目录成为一个独立的 Claude Code 项目，有自己的 `CLAUDE.md` 和 `.claude/` 目录。

---

### Step 2：配置 CLAUDE.md 指向笔记库结构

创建或编辑 `CLAUDE.md`，让 Claude 理解笔记库的组织方式：

```markdown
# Obsidian 知识库助手

## 项目背景
这是一个 Obsidian 笔记库，存放了我的项目文档、学习笔记和会议记录。

## 笔记库结构
.
├── 0- inbox/           # 收集箱，新笔记暂存
├── 1- 项目/            # 各项目相关文档
│   ├── ETF研究/        # ETF 量化研究项目
│   ├── 小红书运营/     # 内容运营项目
│   └── AI工具测评/     # 工具研究
├── 2- 学习/            # 知识沉淀
│   ├── 编程/           # 编程语言学习
│   ├── 金融/           # 金融知识
│   └── AI/             # AI/ML 相关
├── 3- 会议/            # 会议记录
└── 4- 归档/            # 已完成/归档

## 笔记规范
- 文件名格式：YYYY-MM-DD-标题.md
- 使用双向链接：[[页面名]]
- 标签格式：#标签名
- 摘要写在文件头部 YAML front matter

## 你的能力
- 读取和理解所有 .md 文件内容
- 根据自然语言问题找到相关笔记
- 生成摘要、标签、知识点关联
- 帮我梳理笔记之间的关联关系

## 使用建议
- 查询前先告诉我你想找什么方向的内容
- 可以让我帮你生成新的笔记草稿
- 可以让我帮你整理和优化现有笔记结构
```

### Step 3：用自然语言查询笔记内容

```bash
# 进入笔记库目录（每次对话）
cd ~/Obsidian/MyVault
claude
```

**示例对话**：

```
用户：帮我找一下最近关于 ETF 择时策略的笔记

Claude：
根据你的笔记库，我找到了以下相关内容：

📄 2024-11-20-ETF择时模型讨论.md
   - 讨论了均线交叉和 RSI 指标的择时效果
   - 结论：均线策略在趋势市中有效，震荡市表现差

📄 2024-12-05-量化策略复盘.md
   - 提到了布林带均值回归策略的回测结果
   - 夏普比率 1.8，最大回撤 12%

📄 2024-12-15-2024年总结与2025计划.md
   - 将「ETF网格策略」列为 2025 主推方向

需要我帮你做进一步分析，或者生成一份汇总报告吗？
```

---

### Step 4：让 Claude 帮你生成内容

```bash
# 帮我生成会议记录摘要
claude "请读取 3-会议/2024-12-20-产品规划会议.md，生成一份结构化的摘要，提取关键决策、行动项和负责人"

# 帮我生成标签和关联
claude "请扫描 1-项目/ETF研究/ 目录下的所有笔记，自动生成标签体系和笔记之间的链接关系"

# 帮我整理归档
claude "请扫描 inbox/ 目录，把超过 30 天未整理的笔记列出来，并给出归档建议"
```

## 进阶配置

### 配置文件（.claude/commands.json）

为常用操作定义快捷命令：

```json
{
  "commands": {
    "summarize": "请读取当前目录下的所有笔记，生成一份知识图谱摘要",
    "weekly": "扫描 inbox/ 和最近一周新增的笔记，生成周报",
    "link": "找出所有笔记中的 [[双向链接]]，生成链接关系图"
  }
}
```

### 多 Vault 管理

如果你有多个 Obsidian 库：

```bash
# Vault A
cd ~/Obsidian/VaultA
claude

# Vault B（另一个独立会话）
cd ~/Obsidian/VaultB
claude
```

每个 Vault 独立配置，互不干扰。

## 适用场景

### ✅ 非常适合
- 个人知识管理（学习笔记、项目文档、读书笔记）
- 会议记录整理与追踪
- 建立个人第二大脑
- 需要在本地保持隐私的知识库

### ⚠️ 限制
- 不支持 Obsidian 插件生态（如 Dataview、 Templater 等插件生成的内容）
- 纯文本 .md 文件支持最好，嵌入的附件/图片无法读取
- 大量笔记（1000+）时建议定期清理，避免上下文溢出

## 常见问题

**Q: Obsidian 有插件可以直接对接 Claude 吗？**
A: Obsidian 的 Local REST API 插件可以暴露笔记内容，但用 Claude Code 直接读取文件更简单直接。

**Q: 笔记太多，Claude 上下文不够怎么办？**
A: 使用文件夹级别切入，每次只让 Claude 聚焦一个子目录；或定期清理过期笔记。

**Q: 能否让 Claude 记住我的笔记写作风格？**
A: 可以，在 CLAUDE.md 中添加「写作风格」章节，描述你偏好的格式和语气。
